# Entities (Οντότητες) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Six SoftOne-aligned entity lists (Supplier, Customer, Product, ProductCategory, Color, Size) with code+name everywhere, CRUD UI at `/entities`, pull-only connector sync (SoftOne + WooCommerce), xlsx import/export, and a new `ENTITY` process-field type referencable from process templates.

**Architecture:** A per-kind registry (`src/lib/entities/registry.ts`) centralizes labels, columns, xlsx headers and prisma delegates so CRUD/xlsx/sync/UI are generic over `EntityKind`. Sync mappers are pure row-mapping functions + thin fetchers. The `ENTITY` field type stores `valueEntityId` (polymorphic, no DB FK) and resolves to «code — name» at read time.

**Tech Stack:** Next.js 16 App Router, Prisma/MySQL (`prisma db push`, NEVER `migrate dev`), shadcn/ui, exceljs (already a dependency), vitest.

**Spec:** `docs/superpowers/specs/2026-07-12-entities-design.md`

**Conventions (verified in codebase):**
- Guard pattern: `auth()` → redirect `/auth/login`; `requireRole(session.user.role, [Role.SUPER_ADMIN, Role.ADMIN])` → redirect `/dashboard` (see `src/app/(app)/settings/connectors/page.tsx`, but with both roles).
- Server actions return `{ ok: true, ... } | { ok: false, error: string }`, Greek messages.
- Toast mechanism: inline `Msg` banner pattern from `connectors-client.tsx` (no sonner).
- Field-type plumbing lives in `src/lib/process-fields/field-types.ts` (FIELD_TYPES, LABELS, valueColumnFor), `coerce.ts`, `pivot.ts`, `pivot-view.ts`; inputs in `src/components/process-fields/dynamic-field-input.tsx` + `task-fields-form.tsx`; template editor in `src/app/(app)/process-templates/wizard/step-fields.tsx`; field persistence in `process-templates/actions.ts` (FieldInput type + create/update).
- Connector access: `getConnectorValues(type)` from `src/lib/connectors/read.ts` (server-only, decrypted secrets). Woo keys: `config.baseUrl`, `secrets.consumerKey`, `secrets.consumerSecret`. SoftOne keys: read `src/lib/connectors/registry.ts` for the SOFTONE field names (serial/URL, username, password, appId, company, branch, module, refid — use the exact keys defined there).
- SoftOne responses are win1253 (+possibly gzip): decode via `iconv.decode(Buffer.from(await res.arrayBuffer()), "win1253")`, gunzip first if `content-encoding: gzip`. `iconv-lite` — check package.json; if absent, `npm install iconv-lite`.
- Tests colocated `src/lib/**/*.test.ts`, `npm test` (vitest).

Shape (implement fully):

```ts
import type { EntityKind } from "@prisma/client";

export const ENTITY_KINDS: EntityKind[] = [
  "SUPPLIER", "CUSTOMER", "PRODUCT", "PRODUCT_CATEGORY", "COLOR", "SIZE",
];

export type EntityColumn = {
  key: string;              // prisma field name
  headerGr: string;         // xlsx/table header, "*" suffix means required
  kind: "string" | "number" | "boolean";
  required?: boolean;
};

export type EntityMeta = {
  kind: EntityKind;
  labelGr: string;          // «Προμηθευτές» κλπ.
  labelSingularGr: string;
  prismaModel: "supplier" | "customer" | "product" | "productCategory" | "color" | "size";
  columns: EntityColumn[];  // ordered; first two always code,name
};
```

Columns per kind: base `code`(required) + `name`(required) + `isActive`(boolean, default true); SUPPLIER/CUSTOMER add afm/address/city/zip/phone/email; PRODUCT adds priceWholesale/priceRetail/vatPct/unit (number/number/number/string) — categoryId/colorId/sizeId are NOT xlsx columns in v1 (linked via UI/sync only). `entityMeta(kind)`, `xlsxHeadersFor(kind)` (headerGr + "*" when required), `rowFromRecord(kind, rec)` → `(string|number|boolean|null)[]`, `recordFromRow(kind, row)` → validated record (throws Greek `Error` on missing code/name; trims strings; parses numbers with comma-or-dot decimal; isActive accepts `true/false/1/0/ΝΑΙ/ΟΧΙ` case-insensitive, default true when empty).
